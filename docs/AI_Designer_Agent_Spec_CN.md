## 1. 概述

### 1.1 目的

AI Designer Agent 是一个智能可视化工具，使户外生活行业的专业人士能够在客户家中现场**秒级生成逼真的设计渲染图**。该 Agent 将庭院照片转化为令人惊艳的产品可视化效果（阳光房、ADU、凉亭、拉链式百叶窗），从而加速签单成交。

### 1.2 目标用户
- 户外生活行业的中小企业主
- 进行现场咨询的销售代表
- 向客户展示方案的设计顾问

### 1.3 核心价值主张
- **产出导向**：通过专业可视化效果在现场 <15 分钟内完成签单
- **差异化优势**：现场即时签单能力 vs. 竞争对手的 24-48 小时交付周期

### 3.3 设计赋能全矩阵（新增 — 基于 Larry 反馈 2026-03-15）

<mark>**设计不仅是效果图**</mark>。AI Designer 应逐步输出完整的设计赋能矩阵：

| 需求 ID | 设计产出 | 用途 | 面向对象 | 优先级 |
|---------|---------|------|---------|--------|
| FR-034 | **实景融合效果图** | 客户直观看到产品在自家院子的效果 | B→C 展示 | P0 |
| FR-035 | **Site Plan 总平面图** | 产品位置、尺寸、与主屋/院子关系 | B→C 展示 + 合规申请 | P0 |
| FR-036 | **产品内部平面布局图** | 阳光房内部家具布置、功能区划分 | B→C 使用建议 | P1 |
| FR-037 | **产品立面图** | 各面外观和尺寸标注 | 合规申请 + 生产 | P1 |
| FR-038 | **Before-After 动画** | 旧庭院→新产品的动态转变 | B→C 营销 | P1 |
| FR-039 | **结构计算书** | 基础承重、风荷载等工程数据 | 合规申请（持牌工程师签章） | P2 |
| FR-040 | **基础选型建议图** | 地基类型和施工方案 | 施工指导 | P2 |

### 3.4 测量能力（新增 — 基于 Larry 反馈 2026-03-15）

<mark>**测量是经销商首要痛点**</mark>，传统方式需要初测和复核两次上门。AI Designer 的测量能力分阶段演进：

| 需求 ID | 阶段 | 能力 | 精度 | 状态 |
|---------|------|------|------|------|
| FR-041 | MVP | 照片上传 + AI 视觉估算尺寸 | ±10-15% | 当前 |
| FR-042 | 近期 | 激光三维扫描集成（UNRE/Leica） | 毫米级 | 规划中 |
| FR-043 | 未来 | AI + 激光融合，一次上门完成初测+复核 | 毫米级 + 自动校验 | 远期 |

## 5. UI/UX 规格

### 5.1 布局
- **桌面端**：双栏网格（左侧输入，右侧输出）
- **移动端**：堆叠布局（上方输入，下方输出）
- **断点**：1024px 处切换 lg:grid-cols-2

### 5.2 配色方案
- 主色调：蓝色 (#3B82F6)
- 成功：绿色 (#10B981)
- 警告：琥珀色 (#F59E0B)
- 错误：红色 (#EF4444)
- 背景：白色 (#FFFFFF) / 灰-50 (#F9FAFB)

### 5.3 字体排版
- 字体：Roboto（system-ui 回退）
- 标题：font-bold, text-2xl
- 正文：font-normal, text-sm
- 标签：font-medium, text-gray-700

### 5.4 组件
- 步骤指示器：圆形编号徽章（1-4）
- 选择卡片：边框高亮 + 背景色调
- 按钮：圆角-lg，主要操作使用渐变
- 卡片：圆角-xl，border-gray-200，shadow-sm

## 6. 数据模型

### 6.1 设计会话
```typescript
interface DesignSession {
  id: string;
  projectId: string;
  yardPhoto: string; // URL 或 base64
  productType: 'sunroom' | 'adu' | 'pergola' | 'blinds';
  style: 'modern' | 'traditional' | 'rustic';
  frameColor: 'black' | 'white' | 'bronze' | 'gray' | 'forest';
  size: 'small' | 'medium' | 'large';
  generatedDesigns: DesignOutput[];
  selectedVariant: number;
  createdAt: Date;
  createdBy: string;
}
```

### 6.2 设计产出
```typescript
interface DesignOutput {
  id: string;
  imageUrl: string;
  beforeImageUrl?: string;
  estimatedSqft: number;
  estimatedPrice: number;
  estimatedTimeline: string;
  complianceStatus: ComplianceCheck[];
  // 新增：设计赋能全矩阵产出
  sitePlanUrl?: string;       // Site Plan 总平面图
  interiorLayoutUrl?: string; // 产品内部布局图
  elevationUrl?: string;      // 产品立面图
  animationUrl?: string;      // Before-After 动画
  structuralCalcUrl?: string; // 结构计算书
  foundationGuideUrl?: string;// 基础选型建议图
}
```

### 6.3 测量数据（新增）
```typescript
interface MeasurementData {
  id: string;
  projectId: string;
  method: 'photo_ai' | 'laser_scan' | 'manual';
  dimensions: {
    width: number;
    depth: number;
    height: number;
    unit: 'ft' | 'in' | 'mm';
  };
  accuracy: string; // e.g., "±10%" or "±2mm"
  scanDate: Date;
  rawScanDataUrl?: string; // 激光扫描原始数据
  verificationStatus: 'pending' | 'verified' | 'flagged';
}
```

## 7. API 端点

### 7.1 设计生成
```
POST /api/v1/designs/generate
请求: { projectId, yardPhoto, productType, preferences }
响应: { designId, status, estimatedTime }
```

### 7.2 设计状态
```
GET /api/v1/designs/{designId}/status
响应: { status, progress, previewUrl? }
```

### 7.3 设计变体
```
POST /api/v1/designs/{designId}/variants
响应: { variants: [{ id, imageUrl }] }
```

### 7.4 设计赋能全矩阵（新增）
```
POST /api/v1/designs/{designId}/full-matrix
请求: { outputs: ['site_plan', 'interior_layout', 'elevation', 'animation'] }
响应: { matrixId, status, outputs: [{ type, url, status }] }
```

### 7.5 测量数据（新增）
```
POST /api/v1/measurements/upload
请求: { projectId, method, rawData | dimensions }
响应: { measurementId, dimensions, accuracy, verificationStatus }
```

### 8.1 内部集成

|------|---------|------|

## 12. 风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| AI 生成质量不稳定 | 高 | 实施质量评分，增加人工审核选项 |
| 生成时间过长 | 中 | 异步处理 + 通知机制 |
| 照片质量参差不齐 | 中 | 提供拍照指南，自动增强处理 |
| 客户隐私顾虑 | 高 | 明确的授权流程，安全加密存储 |
| **测量精度不足（MVP 阶段）** | 中 | 明确告知估算精度范围，引导用户确认或修正 |
| **设计全矩阵产出过多导致生成延迟** | 中 | 按优先级分批生成，P0 即时输出，P1/P2 异步处理 |
