## 1. 产品概述

### 1.1 平台定位
Nestopia B2B 运营平台是管理户外生活空间项目全生命周期的内部指挥中心——从客户初始意向到安装验收及售后支持。平台整合了项目管理、AI 驱动设计工具、合规工作流和商业智能分析于统一仪表板。

### 1.2 目标用户

| 角色 | 描述 | 主要任务 | 使用频率 |
|------|------|---------|---------|
| **运营管理员** | 日常业务管理者 | 监控 KPI、管理订单、统筹项目 | 每日 |
| **项目经理** | 负责单个项目交付 | 跟踪工作流步骤、管理风险/问题、协调团队 | 每日 |
| **销售代表** | 处理客户咨询与转化 | 查看客户、创建报价、跟踪订单 | 每日 |
| **设计师** | 使用 AI 工具进行概念和详细设计 | 生成设计、审核测量数据、准备效果图 | 按项目 |
| **合规专员** | 确保满足法规要求 | 审核许可证、HOA 审批、检验报告 | 按项目 |
| **公司负责人 / 高管** | 战略决策层 | 收入仪表板、团队绩效、项目组合健康度 | 每周 |

### 1.3 设计原则
```
清晰度 > 信息密度 > 美观度 > 动效
```
- **清晰度**: 每个界面都能回答"我下一步该做什么？"
- **信息密度**: 展示足够数据但不过载；渐进式信息呈现
- **美观度**: 专业、高端的视觉风格，与 Nestopia 品牌一致
- **动效**: 有目的的过渡动画，不做纯装饰性动效

### 1.4 设计系统参考
- 色彩体系: `#222222` (主色深灰黑), `#f5f5f5` (背景色), 语义化状态色
- 字体: Roboto 400/500/600/700
- 间距: 4px 基础单位 (Tailwind 比例)
- 圆角: 8px (卡片), 12px (弹窗), full (徽标)
- 详见 `UI_DESIGN_SYSTEM.md` 完整令牌参考

> **v1.2.0 重大重构**: 导航架构从"按功能分区"转为"以项目为中心"。
> 项目列表提升至侧边栏一级导航；所有 Agents（含 Chatbot）下沉至项目工作台内上下文使用。

#### 交互设计

| 操作 | 行为 | 动效 |
|------|------|------|
| 点击概要卡片 | 展开下方详情区域；高亮卡片边框 | 平滑下滑 (200ms)，边框颜色变化 |
| 再次点击同一卡片 | 收起详情区域 | 上滑收起，边框恢复 |
| 点击不同卡片 | 切换详情区域 | 交叉淡入淡出 |
| 营收卡片 | 不可点击；展示迷你柱状趋势图 | 静态 |

#### 卡片状态
- **默认态**: `border-gray-200`，轻微悬停阴影
- **激活/展开态**: `border-{color}-400`，增强阴影，箭头旋转 180 度
- **悬停态**: `border-{color}-400` 提示，轻微阴影提升

#### 数据密度指南
- 概要卡片: 1 个主要指标 + 2 个次要统计
- 订单表格: 最多 6 列 (编号、客户、产品、金额、状态、日期)
- 客户表格: 6 列 (姓名、邮箱、地址、项目数、消费总额、状态)
- 产品网格: 卡片布局含图标、名称、分类、价格、状态徽标

### 3.2 项目工作台 (v1.2.0 重构)

> **重构说明**: v1.2.0 将项目管理从"主从详情"布局重构为"侧边栏项目列表 + 全宽项目工作台"。
> 项目列表从主内容区移至侧边栏；Agents 从侧边栏移至项目详情页内水平工具栏。

#### 设计理念
用户的日常工作以**项目为中心**——每天处理的是 Johnson 家的阳光房、Martinez 家的 ADU。
Agents 是服务于项目的**工具**，不是独立入口。将 Agents 嵌入项目工作台，实现：
- **零切换成本**: 在同一项目上下文中直接调用设计、报价、合规等工具
- **自动关联项目**: Agent 打开时自动加载当前项目数据，无需手动选择项目
- **聚焦工作流**: 侧边栏只展示项目列表，减少认知负担

#### 3.2.4 Agent 工作面板交互

| 操作 | 行为 | 动效 |
|------|------|------|
| **点击 Agent 卡片** | 展开对应 Agent 工作面板 (在工具栏下方); 高亮卡片为激活态 | `slideDown 0.3s ease-out`, 面板 `max-height: 0 → auto` |
| **再次点击同一 Agent** | 收起工作面板; 卡片恢复默认态 | `slideUp 0.2s ease-in` |
| **点击不同 Agent** | 交叉切换: 收起当前面板 → 展开新面板; 更新激活态 | 交叉淡入淡出 `0.25s` |
| **面板内操作** | Agent 自动绑定当前项目上下文 (无需手动选择项目) | — |
| **关闭面板** | 面板右上角 ✕ 按钮, 或点击已激活的 Agent 卡片 | `fadeOut + slideUp` |

**关键 UX 改进 (vs v1.1.0)**:
- Agent 页面不再需要"选择项目"下拉框 — 项目已由侧边栏选择确定
- Agent 面板内容与旧版 Agent 独立页面一致，但容器变为可展开/收起面板
- 多个 Agent 可以快速切换，无需离开项目页面

- 表头: `bg-gray-50/50`，大写 text-xs，tracking-wider
- 数据行: `border-b border-gray-50`，悬停 `bg-gray-50/50`
- 操作列: 图标按钮 (省略号或特定图标)
- 空状态: 居中消息含图标
- 加载状态: 骨架屏闪烁行

### 4.4 卡片设计
- 背景: 白色
- 边框: `border-gray-200` (1px), `rounded-xl` (12px)
- 内边距: `p-5` (20px)
- 悬停: `shadow-lg` + 边框颜色变化
- 卡片内区块头: `bg-gray-50/70`，图标 + 文字，`border-b`

### 4.5 表单输入
- 边框: `border-gray-200`, `rounded-lg`
- 聚焦: `ring-2 ring-gray-300` (中性) 或 `ring-{color}-300` (上下文)
- 标签: `text-sm font-medium text-gray-700`
- 辅助文字: `text-xs text-gray-400`
- 错误: `border-red-300 ring-red-300`，红色辅助文字

### 4.6 空状态
所有空状态遵循以下模式:
```
[图标 - 5xl, gray-400]
[标题 - lg, font-medium]
[副标题 - sm, gray-400]
[可选 CTA 按钮]
```

### 4.7 加载状态
- 初始加载使用骨架屏闪烁
- 操作中使用内联加载器 (按钮 → 加载状态)
- 多步骤操作使用进度条

## 7. 无障碍访问

### 7.1 要求
- WCAG 2.1 AA 级合规目标
- 所有交互元素支持键盘导航
- 文字颜色对比度 ≥ 4.5:1
- 所有纯图标按钮配有屏幕阅读器标签
- 所有交互元素有焦点指示器

### 7.2 实现说明
- 所有 `<button>` 和 `<a>` 元素需有描述性文字或 `aria-label`
- 状态颜色始终配合文字标签 (不仅依赖颜色)
- 表格包含 `<thead>` 语义表头
- 表单输入关联 `<label>` 元素

## 9. 设计决策日志

| 日期 | 决策 | 原因 |
|------|------|------|
| 2026-03-18 | 将订单/客户/产品合并进公司概览 | 减少侧边栏混乱；这些是概要级视图，非独立工作流 |
| 2026-03-18 | 服务工作流移入项目详情 (不再独立页面) | 工作流始终与项目关联；独立页面冗余 |
| 2026-03-18 | 项目管理采用主从详情布局 | 高效浏览项目；减少页面切换；借鉴邮件客户端 UX |
| 2026-03-18 | 营收卡片为静态 (不可点击) | 营收是 KPI 指标；详细分析计划在第二阶段实现 |
| 2026-03-18 | 智能体和设置保留为独立侧边栏区块 | 智能体页面足够复杂需独立视图；后续会重构 |
| 2026-03-16 | 手动测量为主要方式 (第3步) | 激光三维扫描尚不可行；手动工具可靠且经过验证 |
| 2026-03-24 | Chatbot 采用双模式 UX (浮动气泡 + 全页面) | 浮动气泡适合快速问答不离开当前页；全页面适合深度多轮对话和会话管理 |
| 2026-03-24 | Chatbot 意图识别采用关键词匹配 (MVP) | LLM Function Calling 增加延迟和复杂度；关键词匹配在 MVP 阶段足够准确，Phase 2 升级 |
| 2026-03-24 | Chatbot 作为路由层而非独立 Agent | Chatbot 不持有独立 AI 模型；它串联 5 个 Agent，避免重复建设 |
| 2026-03-24 | 全页面 Chatbot 在导航到时隐藏 FAB | 避免 FAB 遮挡全页面内容；两种模式互斥展示 |
| 2026-03-26 | **项目列表从主内容区迁移至侧边栏** | 用户日常工作以项目为中心，项目切换是最高频操作；侧边栏提供持久可见的项目导航，消除"先进入项目页再选项目"的额外步骤 |
| 2026-03-26 | **Agents 从侧边栏迁入项目工作台水平工具栏** | Agents 是服务于项目的工具，不是独立工作流入口；嵌入项目页实现零切换成本 + 自动项目上下文绑定，无需手动选择项目 |
| 2026-03-26 | **Chatbot 归入项目工具栏 (与其他 5 个 Agent 并列)** | 隐藏复杂性：业务老板不需要理解"5个Agent+1个Chatbot"的架构，只需看到项目和项目里的工具；减少侧边栏导航项数量，降低认知负担 |
| 2026-03-26 | Agent 面板采用展开/收起模式 (非独立页面) | 减少页面切换；用户可在同一视图中快速切换不同 Agent 工具；与项目详情共存而非替代 |
| 2026-03-26 | 项目详情改为全宽布局 (移除左侧项目列表面板) | 项目列表已移至侧边栏，主内容区无需分栏，全宽利用让项目详情和 Agent 面板有更充裕的展示空间 |
