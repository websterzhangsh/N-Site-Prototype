## 1. 架构总览

### 1.1 核心原则：Price-at-Quote（报价锁价）

Nestopia 的定价体系遵循一个核心原则：**报价单创建时锁定价格，后续价目表变动不影响已签发报价**。

```
┌───────────────┐      ┌────────────────┐      ┌───────────────┐      ┌──────────────┐
│  供应商价目表   │─────▶│  平台价目表      │─────▶│  报价单快照     │─────▶│  订单/合同     │
│  (Supplier)   │      │  (Price List)  │      │  (Quotation)  │      │  (Order)     │
│               │      │                │      │               │      │              │
│ · 工厂出厂价   │      │ · COGS 成本价   │      │ · 冻结的单价   │      │ · 永久锁定    │
│ · 阶梯定价     │      │ · 市场价        │      │ · 价目表版本号  │      │ · 不可变更    │
│ · 受控文件     │      │ · 优惠售价      │      │ · 有效期       │      │ · 审计基础    │
└───────────────┘      └────────────────┘      └───────────────┘      └──────────────┘
     外部输入                内部计算                 时间点快照               法律约束
```

### 1.2 参与者与职责

| 角色 | 职责 | 操作频率 |
|------|------|---------|
| **供应商** | 签发出厂价目表（2026-H005 等） | 半年~年度更新 |
| **运营管理** | 设定业务参数（加价率、折扣、物流比） | 季度调整 |
| **销售/客服** | 创建报价单、调整个案折扣 | 每日 |
| **系统** | 自动计算、快照保存、过期提醒 | 实时 |

## 2. 定价链四层模型

### 2.1 分层结构

```
┌─────────────────────────────────────────────────────────────────┐
│  L0  供应商价（Supplier Price）                                    │
│      来源: 2026-H005 报价表 PDF / Excel                           │
│      存储: js/data/pricing-data.js → zbSKUCatalog[].priceTiers   │
│      性质: 工厂出厂价（RMB/㎡），纯成本基础                          │
├─────────────────────────────────────────────────────────────────┤
│  L1  经销商成本（COGS = Cost of Goods Sold）                       │
│      公式: 供应商价 × 折扣率 + 物流清关费 + 安装费                    │
│      参数: supplierDiscountRate(0.9) + shippingCostRate(0.3)      │
│           + installationFeePerSqm(191)                           │
│      存储: 实时计算，不持久化                                        │
├─────────────────────────────────────────────────────────────────┤
│  L2  消费者售价（Selling Price = Preferential Price）               │
│      公式: COGS × marketMarkup(2.92) × preferentialDiscount(0.5)  │
│      用途: 对外报价基准（面向终端消费者）                              │
│      存储: 实时计算 → 写入报价单时快照冻结                            │
├─────────────────────────────────────────────────────────────────┤
│  L3  报价单价格（Quoted Price = Snapshot）                          │
│      性质: 时间点快照，创建后不随价目表变动                            │
│      属性: unitPrice + priceListVersion + validUntil              │
│      存储: localStorage + Supabase quotation_data JSONB           │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 完整计算链示例

```
WR110A-63 | 面积 6㎡ | MK-45 电机

L0: 供应商价 = 295 元/㎡ (priceTiers: ≤6㎡档)
L1: COGS    = 295×0.9 + 295×0.9×0.3 + 191 = 265.5 + 79.65 + 191 = 536.15 元/㎡
L2: 售价    = 536.15 × 2.92 × 0.50 = 782.78 元/㎡
L3: 报价单  = 782.78 × 6㎡ = 4,696.68 元 (帘面)
           + MK-45: 355 × 1.13 = 401.15 元 (驱动)
           = ¥5,097.83 (冻结在报价单中)
```

## 3. SKU 价格生命周期

### 3.1 状态流转

```
┌─────────┐     ┌──────────┐     ┌─────────┐     ┌──────────┐
│ DRAFT   │────▶│ ACTIVE   │────▶│ SUNSET  │────▶│ ARCHIVED │
│ 草稿     │     │ 生效中    │     │ 过渡期   │     │ 归档      │
└─────────┘     └──────────┘     └─────────┘     └──────────┘
                     │                                  ▲
                     │   新版本发布时                      │
                     └──────────────────────────────────┘
```

| 状态 | 含义 | 对报价的影响 |
|------|------|------------|
| **DRAFT** | 新价格已录入但未生效 | 不可用于新报价 |
| **ACTIVE** | 当前有效价目表 | 新报价使用此价格 |
| **SUNSET** | 已被新版替代，30天过渡期 | 已有报价继续有效，新报价使用新价格 |
| **ARCHIVED** | 历史版本，仅供查阅 | 旧报价仍引用此版本号 |

### 3.2 价目表版本化

每次价格变动都产生一个新版本：

```javascript
// pricing-data.js 价目表元数据
zbPricingMeta: {
    version: '2026-05-08-v1',         // 当前版本标识
    effectiveDate: '2026-05-08',      // 生效日期
    previousVersion: '2026-04-15-v2', // 上一版本（可追溯）
    source: '2026-H005 签发版',        // 数据来源
    changedSKUs: ['WR110A-63'],       // 本次变更涉及的 SKU
    changelog: '调整 WR110A-63 ≤6㎡ 档位 295→310'
}
```

### 3.3 SKU 价格变更流程

```
Step 1: 供应商签发新价目表
    ↓
Step 2: 运营确认变更范围 → 更新 pricing-data.js
    ↓
Step 3: 递增 zbPricingMeta.version
    ↓
Step 4: 已保存报价单不受影响（快照机制）
    ↓
Step 5: 下次打开旧报价单时 → 显示价格变动提示（可选更新或保留）
```

## 4. 报价单价格快照机制

### 4.1 当前实现（v3.0）

报价单保存时，**unitPrice 被硬写入 JSON**，形成价格快照：

```javascript
// quotation-editor.js → saveQuotation()
quotation = {
    id: 'Q-1715000000000',
    savedAt: '2026-05-08T10:30:45.000Z',
    lineItems: [{
        product: 'WR110A-63 ...',   // 产品显示名
        width: 3000,                 // mm
        height: 2500,                // mm
        unitPrice: 782.78,           // ★ 快照价格 — 保存时冻结
        qty: 1
    }],
    accessories: [{
        name: 'MK-45 ...',
        unitPrice: 401.15,           // ★ 快照价格
        qty: 1
    }]
};
```

### 4.2 目标结构（增强版）

```javascript
quotation = {
    id: 'Q-1715000000000',
    savedAt: '2026-05-08T10:30:45.000Z',
    priceListVersion: '2026-05-08-v1',   // ★ 基于哪个价目表版本
    validUntil: '2026-06-07',            // ★ 报价有效期（30天）
    status: 'quoted',                     // draft → quoted → expired → accepted
    lineItems: [{
        product: 'WR110A-63 ...',
        skuKey: 'WR110A-63',             // ★ SKU 标识（用于价格对比）
        width: 3000,
        height: 2500,
        unitPrice: 782.78,               // 快照价格
        priceTier: '≤6m²',              // ★ 命中的价格梯度
        qty: 1
    }],
    accessories: [{
        name: 'MK-45 ...',
        driveKey: 'MK-45',              // ★ 驱动系统标识
        unitPrice: 401.15,
        qty: 1
    }]
};
```

### 4.3 三个关键行为规则

| 场景 | 行为 | 原因 |
|------|------|------|
| **保存报价单** | 冻结当前 unitPrice + 记录 priceListVersion | 价格快照不可变 |
| **加载已有报价单** | 恢复快照价格，**不自动更新** | 商业承诺不能单方面变更 |
| **用户切换产品下拉框** | 使用当前最新价格（非快照） | 视为"创建新行项"意图 |

## 5. 价格变更影响分析

### 5.1 问题：修改 SKU 价格后的影响矩阵

| 操作 | 已保存的报价单 | 新创建的报价单 | 正在编辑的报价单 |
|------|-------------|-------------|--------------|
| 修改 `pricing-data.js` 中 SKU 单价 | ❌ 不影响 | ✅ 使用新价格 | ⚠️ 见下文 |
| 修改业务参数（markup 等） | ❌ 不影响 | ✅ 使用新参数 | ⚠️ 见下文 |
| 新增 SKU | ❌ 不影响 | ✅ 可选新 SKU | ⚠️ 刷新后可见 |
| 删除 SKU | ❌ 旧报价保留快照 | ❌ 不可选已删 SKU | ⚠️ 已选的保留 |

### 5.2 "正在编辑"的详细行为

用户打开一个已保存的报价单进行修改时：

```
场景 A: 只改数量/尺寸 → 保存
  → unitPrice 保持旧快照值 → 总价按旧价重算 ✅ 合理

场景 B: 切换产品型号（下拉框） → 保存
  → unitPrice 被当前价目表覆盖 ⚠️ 用户可能不知道价格变了

场景 C: 手动修改 unitPrice 输入框 → 保存
  → 用户显式覆盖 → 以用户输入为准 ✅ 合理
```

**场景 B 是当前系统的风险点** — 需要增加价格变动提示。

### 5.3 价格变动感知机制（待实现）

当打开一个旧报价单且检测到价格变化时，应提示用户：

```
┌──────────────────────────────────────────────────────┐
│  ⚠️ Price Change Detected / 检测到价格变动             │
│                                                       │
│  以下产品的当前价格与报价单中的快照价格不同：              │
│                                                       │
│  WR110A-63:  ¥782.78/㎡ → ¥810.50/㎡ (+3.5%)         │
│  MK-45 电机:  ¥401 → ¥420 (+4.7%)                    │
│                                                       │
│  报价单价目表版本: 2026-04-15-v2                        │
│  当前价目表版本:   2026-05-08-v1                        │
│                                                       │
│  [保留原价 Keep Original]   [更新为新价 Update Prices]  │
└──────────────────────────────────────────────────────┘
```

## 7. 演进路线图

### Phase 1: 基础追溯（低侵入，可立即实施）

**目标**: 让每份报价可以追溯到具体的价目表版本

| 任务 | 文件 | 变更 |
|------|------|------|
| pricing-data.js 增加 `zbPricingMeta` | `js/data/pricing-data.js` | 新增 ~10 行 |
| 保存报价时记录 `skuKey` + `priceListVersion` | `js/utils/quotation-editor.js` | 修改 saveQuotation() |
| 向后兼容：旧报价无 `skuKey` 视为 null | — | 符合规则 #1 |

### Phase 2: 用户感知（中等复杂度）

**目标**: 用户打开旧报价时知道价格是否变了

| 任务 | 文件 | 变更 |
|------|------|------|
| 加载报价时对比当前 priceLookup | `js/utils/quotation-editor.js` | 修改 loadQuotation() |
| 价格变动提示 UI（Toast 或 Modal） | `company-operations.html` | 新增 HTML 模板 |
| "保留/更新"双按钮交互 | `js/utils/quotation-editor.js` | 新增函数 |

### Phase 3: 有效期管理（业务规则）

**目标**: 过期报价自动标记，防止使用过旧价格

| 任务 | 说明 |
|------|------|
| 报价单增加 `validUntil` 字段 | 默认 savedAt + 30天 |
| 状态流转: `draft → quoted → expired → accepted` | 侧栏/列表显示状态标签 |
| 过期报价打开时强制提示 "Requote" | 不允许直接发给客户 |

### Phase 4: 完整价格历史（长期，需 Supabase）

**目标**: 完整审计链，可回答"这个价格什么时候改过"

```sql
-- 未来 Supabase 表结构
CREATE TABLE sku_price_history (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    sku_key TEXT NOT NULL,
    version TEXT NOT NULL,
    effective_date DATE NOT NULL,
    price_tiers JSONB NOT NULL,
    changed_by TEXT,
    change_reason TEXT,
    created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_sku_price_history_sku ON sku_price_history(sku_key, effective_date DESC);
```

```
┌─────────────────────────────────────────────────────────────────┐
│                    定价公式 — 完整计算链                           │
│                                                                  │
│  ① 折扣供应商价 = 供应商单价(元/㎡) × 供应商折扣率(0.9)            │
│  ② 物流清关费   = ①折扣供应商价 × 物流清关比例(0.30)              │
│  ③ 安装费       = 安装费单价(191 元/㎡)                           │
│  ④ COGS(元/㎡)  = ① + ② + ③ + 面料附加费 + 高度附加费            │
│  ⑤ 市场价       = ④COGS × 市场倍率(2.92)                        │
│  ⑥ 优惠价(售价)  = ⑤市场价 × 优惠折扣(0.50)                      │
│  ⑦ 驱动系统售价  = 驱动供应商价 × (1 + 配件加成率 0.13)            │
│                                                                  │
│  ⑧ 计费面积      = max(实际面积, 最低计费面积 3㎡)                 │
│  ⑨ 帘面总售价    = Σ(每洞口计费面积 × 优惠价)                     │
│  ⑩ 驱动总售价    = Σ(驱动系统售价 × 数量)                         │
│  ⑪ 木箱包装费    = 护罩长度 > 2m ? (长度 × 50元/m) : 0           │
│  ⑫ 项目总售价    = ⑨ + ⑩ + ⑪ + Other Items                      │
│  ⑬ 外币报价      = ⑫ / 汇率 (默认 1 SGD = 5.3612 RMB)          │
│                                                                  │
│  毛利率 = (总售价 − 总成本) / 总售价 × 100%                       │
│  目标毛利率 ≈ 30%（参数预置值下的自然结果）                         │
└─────────────────────────────────────────────────────────────────┘
```
